**PRODUCT REQUIREMENTS (based on the latest user specification):**
- **Visual Identity:** Modern, clean, professional, and responsive design with a blue and black color scheme.
- **Core Modules:**
    1.  **Scanning:** Allow students to use their device's camera to scan QR codes and barcodes. The system should decode the information and display it in a user-friendly format (Product, Code, Sector, etc.).
    2.  **QR Code Generator:** An educational tool for students to input logistic information (product name, code, sector, etc.) and generate a corresponding QR code, demonstrating the encoding process.

    4.  **Material Management:** Allow professors to register, edit, and delete materials that can be used in the simulations.
- **User Roles & Authentication:**
    1.  **Student:** A simple login (e.g., name, class, password). Students have a personal dashboard to track their progress, score, and performance feedback.
    2.  **Professor:** A login to access a master dashboard with analytics on class and individual student performance, including graphs, rankings, and detailed activity logs.
- **Gamification & Performance Tracking:**
    - A detailed scoring system based on correct scans, correct logistic operation choice, time taken, and bonuses for consecutive successes.
    - Students receive a final classification (e.g., Excellent, Good, Regular).
    - Professors can monitor this data to provide pedagogical support.
- **Educational Content:**
    - Dedicated pages explaining What the student learns (concepts like traceability, automation, etc.) and How the interaction works (the step-by-step flow of using the app).
- **Technical Features:**
    - The system should include a Demo Mode with pre-populated data for easy classroom presentation.
    - Users should be able to export reports/history to PDF and CSV.
